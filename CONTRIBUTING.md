# 🤝 Contributing to Visitfy Website

Thank you for your interest in contributing! Since this is a proprietary project, contributions are by invitation or explicit permission only.

---

## 📋 Before You Start

* Read the [LICENSE](LICENSE) — this is a proprietary codebase
* For any contribution, please [open an issue](../../issues) first to discuss what you'd like to change
* All contributors must follow the [Code of Conduct](CODE_OF_CONDUCT.md)

---

## 🐛 Reporting Bugs

Use the **Bug Report** issue template and include:

* A clear description of the problem
* Steps to reproduce it
* Expected vs. actual behavior
* Browser, OS, and PHP version if relevant
* Screenshots if helpful

---

## 💡 Suggesting Features

Use the **Feature Request** issue template and include:

* A clear description of the proposed feature
* The problem it solves or the value it adds
* Any relevant examples or references

---

## 🔧 Submitting Changes

If you have been granted permission to contribute:

1. Fork the repository
2. Create a new branch: `git checkout -b feature/your-feature-name`
3. Make your changes — keep them focused and minimal
4. Test your changes locally (`php -S localhost:8000`)
5. Commit with a clear message: `git commit -m "feat: describe your change"`
6. Push and open a Pull Request using the provided template

---

## ✅ Code Style

* PHP: follow existing patterns (no frameworks, procedural style)
* CSS: add to `assets/css/style.css`, grouped by section
* JS: vanilla only, no build tools required
* Keep changes small and reviewable
