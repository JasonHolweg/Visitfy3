# 🔒 Security Policy

## Supported Versions

| Version | Supported |
| ------- | --------- |
| Latest  | ✅ Yes     |
| Older   | ❌ No      |

Only the latest version of the Visitfy website is actively maintained and receives security updates.

---

## 🚨 Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

If you discover a security vulnerability, please report it responsibly by emailing:

📧 [info@visitfy.de](mailto:info@visitfy.de)

Include in your report:

* A description of the vulnerability
* Steps to reproduce or a proof of concept
* Potential impact (what an attacker could achieve)
* Any suggested fix if you have one

---

## ⏱️ Response Timeline

* **Acknowledgement:** within 48 hours
* **Assessment:** within 5 business days
* **Fix & disclosure:** as soon as a patch is ready

We appreciate responsible disclosure and will credit reporters where appropriate.

---

## 🛡️ Scope

Security issues we care about include, but are not limited to:

* Remote code execution
* SQL or command injection
* Cross-site scripting (XSS)
* Authentication or session vulnerabilities
* Sensitive data exposure
* CSRF vulnerabilities

---

## ⚠️ Out of Scope

* Issues in third-party services (Mailgun, Cloudflare Turnstile, GSAP CDN)
* Theoretical vulnerabilities without a realistic attack scenario
* Issues only reproducible with admin access
